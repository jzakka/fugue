# PRD: Fugue

---

## 1. Summary

Fugue는 크로스미디어 창작물 큐레이션 플랫폼이다. Pinterest가 이미지에 한정된다면, Fugue는 음악, 일러스트, 영상, 글, 코드, 3D, 게임 등 모든 분야의 창작물을 분야를 넘나들며 한곳에서 발견하고 큐레이션한다.

- 외부 창작물 URL을 "핀"하면 OG 메타데이터로 자동 프리뷰 생성
- 핀을 보드로 묶어 주제별 컬렉션 구성
- 클릭/핀 행동 기반으로 취향을 학습하여 개인화된 피드 제공
- 작품 상세에서 연관 작품 자동 추천

## 3. Background

### 3.1 Context

창작물은 플랫폼별로 흩어져 있다. 음악은 SoundCloud, 일러스트는 pixiv, 영상은 YouTube, 코드는 GitHub. 각 플랫폼은 자기 분야만 다루기 때문에, "이 음악에 어울리는 일러스트"나 "이 게임에 맞는 BGM"을 발견하는 방법이 없다.

기존 큐레이션 서비스(Pinterest, Are.na 등)는 주로 이미지에 특화되어 있어, 음악이나 코드 같은 미디어를 동등하게 다루지 못한다.

### 3.2 Why now?

- 인디/취미 창작 시장이 지속 성장 중 (VTuber, 인디 게임, 동인 음악 등)
- 창작물이 흩어져 있는 플랫폼이 계속 늘어남 (SoundCloud, Bandcamp, itch.io, Sketchfab...)
- Pinterest는 이미지 중심. Are.na는 니치. 분야를 넘나드는 크로스미디어 큐레이션은 빈 공간.

### 3.3 레퍼런스

- **Pinterest**: 이미지 큐레이션의 교과서. 핀 + 보드 + 추천 피드 모델.
- **Are.na**: 리서치/아카이빙 목적의 큐레이션. 크리에이티브 니치에 인기.
- **Piapro**: 창작물 투고 + 라이선스 공유. 보컬로이드 생태계 한정.
- **Raindrop.io**: 북마크 매니저. 큐레이션보다 정리 도구에 가까움.

## 4. Objective

### 4.1 목표

분야에 관계없이, 웹에 흩어진 모든 창작물을 한곳에서 발견하고 큐레이션할 수 있게 한다.

| KR3 | 가입 유저 수 | 50명 |
| KR4 | 추천 피드 → 작품 상세 클릭 전환 | 20% |

## 5. Market Segment

### 5.1 타겟 유저

**다양한 분야의 창작물을 탐색하고 모으는 것을 즐기는 사람**

| 특성 | 설명 |
|------|------|
| 동기 | 영감을 위한 수집, 레퍼런스 정리, 취향 큐레이션 |
| 분야 | 음악, 일러스트, 영상, 글, 코드, 3D, 게임, 사운드 등 분야 무관 |
| 현재 행동 | 각 플랫폼 개별 탐색, 브라우저 북마크, Notion/Are.na에 링크 수집 |
| 불만 | 플랫폼별로 흩어짐, 분야 간 크로스 탐색 불가, 북마크는 죽은 링크 |

### 5.2 초기 니치

**음악 MV 제작에 관심있는 크리에이터** — 작곡가가 일러스트를 찾고, 일러스트레이터가 BGM을 찾는 크로스미디어 탐색 패턴이 가장 명확한 그룹.

### 5.3 제외

- 파일 호스팅/저장이 필요한 유저 (Fugue는 링크 큐레이션)
- 유료 커미션/의뢰 목적 (→ VGen, Fiverr)

## 6. Value Proposition

### 6.1 해결하는 문제

> "SoundCloud에서 좋은 곡을 발견했는데, 이 분위기에 맞는 일러스트는 어디서 찾지?"

| 기존 방법 | 문제 |
|-----------|------|
| 각 플랫폼 개별 탐색 | 분야 간 크로스 탐색 불가 |
| 브라우저 북마크 | 정리 안 됨, 프리뷰 없음, 공유 불가 |
| Notion/Are.na | 이미지 위주, 음악/영상 임베드 약함 |
| Pinterest | 이미지만. 음악/코드 핀 불가 |

### 6.2 Fugue가 제공하는 가치

| 가치 | 설명 |
|------|------|
| **크로스미디어 핀** | 음악, 일러스트, 영상, 코드 등 어떤 URL이든 핀 가능 |
| **OG 자동 프리뷰** | URL 붙여넣기만으로 제목/썸네일/설명 자동 생성 |
| **보드 정리** | 핀을 주제별로 묶어 컬렉션으로 관리 |
| **취향 기반 추천** | 클릭/핀 행동을 학습해 개인화된 피드 제공 |
| **연관 작품 발견** | 작품 상세에서 유사한 다른 작품 자동 추천 |

### 6.3 경쟁 대비 차별점

```
              이미지만           크로스미디어
                │                    │
  추천 있음   Pinterest             ▶ Fugue ◀
                │                    │
  추천 없음   Are.na, Raindrop     (없음)
```

## 7. Solution

### 7.1 User Flow

```
[소셜 로그인 (Google/Discord)]
     │
     ▼
[피드] ◄──────────────────────────────────┐
  추천 기반 작품 목록 (분야 필터)           │
     │                                    │
     ▼                                    │
[작품 상세]                                │
  OG 프리뷰 + 태그 + 핀한 유저 수          │
  연관 작품 목록 (하단)                    │
     │                                    │
     ├── [핀하기] → 보드 선택 → 저장 ──────┘
     │
     └── [연관 작품 클릭] → 작품 상세 (반복)

[핀 추가] (별도 진입)
  URL 입력 → OG 프리뷰 → 분야/태그 선택 → 보드 선택 → 저장

[내 프로필]
  닉네임 + 아바타
  내 보드 목록
  전체 핀 목록
```

### 7.2 Key Features

#### Feature 1: 핀 (Pin)

외부 창작물 URL을 큐레이션하는 핵심 단위. 소유권 주장이 아니라 큐레이션.

**입력**
| 필드 | 필수 | 설명 |
|------|------|------|
| 작품 URL | O | 어떤 URL이든 가능 |
| 분야 | O | 도메인 기반 자동감지 + 수동 선택 |
| 스타일 태그 | O | 최소 1개 ~ 최대 5개, 태그당 최대 30자 |

**동작**
- URL 입력 시 서버 사이드 OG fetch → 제목, 썸네일, 설명 자동 추출
- OG fetch 실패 시 수동 입력 폼으로 폴백
- 도메인 자동감지: soundcloud.com → 음악, pixiv.net → 일러스트, youtube.com → 영상, github.com → 코드
- Create, Read, Delete만. Update 없음 (삭제 후 재핀)
- URL 유니크 제약 없음 (여러 사용자가 같은 작품 핀 가능)

#### Feature 2: 보드 (Board)

핀을 주제별로 묶는 컬렉션. Pinterest 보드와 동일한 컨셉.

- 보드 이름 + 설명 (선택)
- 보드에 핀 추가/제거
- 공개/비공개 설정 (비공개 = 본인만 접근)
- 한 핀은 여러 보드에 속할 수 있음

#### Feature 3: 추천 기반 피드 (Recommendation Feed)

사용자 취향 기반 작품 추천. 휴리스틱으로 시작하여 점진적으로 ML로 발전.

**진화 로드맵:**
- **v1 (MVP)**: 태그 빈도 기반 휴리스틱. 내 핀의 태그 분포 → 같은 태그의 다른 작품 추천
- **v2**: 피처스토어 도입. 유저 피처(태그 분포, 분야 선호, 활동 패턴) + 작품 피처(태그, 핀 수, 분야) 관리
- **v3**: ML 모델. 피처스토어 기반 학습 → 개인화 추천

**v1 상세:**
- 내 핀의 태그 빈도수 집계 → 상위 태그와 겹치는 작품을 가중 점수순 정렬. 이미 핀한 작품 제외.
- 추천은 배치/Redis 캐싱 (요청마다 계산하지 않음)
- 콜드 스타트: 핀 10개 미만이면 100% 최신순, 10개 이상이면 50% 추천 + 50% 최신

#### Feature 4: 암묵적 취향 학습 (Implicit Signals)

유저의 클릭/조회 행동을 기록하여 추천에 활용.

- 작품 상세 조회, 핀 생성, 보드 추가 시 이벤트 기록
- `interactions` 테이블: (user_id, work_id, type, created_at)
- type: 'view', 'pin', 'board_add'
- 이 데이터는 v1에서는 추천 보조 시그널, v2+에서는 ML 학습 데이터로 활용

#### Feature 5: 연관 작품 (Related Works)

작품 상세 페이지 하단에 유사 작품 표시.

- v1 로직: 같은 태그를 가진 작품 중 태그 일치순 정렬
- 같은 분야 우선, 크로스 분야도 표시 (태그가 겹치면)
- 최대 10개

#### Feature 6: 인증 (Auth) — 이미 구현됨

- Google OAuth, Discord OAuth
- JWT 기반 인증 (access + refresh token)
- 계정 병합: 같은 이메일이면 자동 병합

### 7.3 Assumptions

| 가정 | 리스크 | 검증 방법 |
|------|--------|----------|
| 크로스미디어 큐레이션에 대한 수요가 있다 | 사용자 부재 | 초기 니치(음악 MV 커뮤니티)에서 검증 |
| OG 메타데이터로 충분한 프리뷰가 가능하다 | 플랫폼별 OG 제한 | 수동 입력 폴백 제공 |
| 태그 기반 휴리스틱 추천이 유용하다 | 추천 품질 | 추천→클릭 전환율 측정 |
| 보드가 핀 정리에 충분한 구조를 제공한다 | UX 복잡도 | 사용 패턴 관찰 |
