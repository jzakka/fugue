# PRD: Fugue

---

## 1. Summary

Fugue는 취미/인디 창작자들이 자신의 작품 링크를 투고하고, 만들고 싶은 프로젝트에 어울리는 다른 분야의 작품을 추천받아 협업으로 이어지게 하는 플랫폼이다. 기존 커미션/거래 플랫폼과 달리 **무료 협업**에 집중하며, **사람이 아닌 작품을 매칭**하는 것이 핵심이다.

## 3. Background

### 3.1 Context

개인적으로 작곡한 곡을 YouTube MV로 만들고 싶지만, 일러스트와 영상 편집 역량이 없다. 이런 상황은 비단 음악에만 국한되지 않는다. 게임을 만들고 싶은 개발자에게는 BGM과 일러스트가 필요하고, 웹툰을 그리는 일러스트레이터에게는 스토리 작가가 필요하다.

현재 이런 협업은 주로 Twitter에서 "콜라보 구합니다" 트윗 → DM으로 이루어진다. 트윗은 타임라인에 묻히고, 역할/장르별 검색이 불가능하며, 상대방의 포트폴리오를 확인하려면 여러 링크를 돌아다녀야 한다.

### 3.2 Why now?

- 인디/취미 창작 시장이 지속 성장 중 (VTuber, 인디 게임, 동인 음악 등)
- 기존 플랫폼(VGen, Skeb, ココナラ)은 전부 **유료 커미션** 모델 — 무료 협업 공간은 비어 있음
- Piapro는 특정 생태계에 한정되어 있어 범용 무료 협업 매칭 서비스는 부재

### 3.3 레퍼런스

- **Piapro**: 창작물 투고 + 라이선스 공유. 매칭/추천 기능은 없음
- **VGen**: 서구권 아트 커미션 플랫폼 (5% 수수료, 초대제)
- **Skeb**: 일본 원웨이 커미션 (협상 없는 의뢰 모델)
- **ココナラ**: 일본 스킬 마켓플레이스 (22% 수수료)

## 4. Objective

### 4.1 목표

취미 창작자가 자신의 작품을 기반으로, 협업에 필요한 다른 분야의 작품과 창작자를 가장 쉽게 발견할 수 있게 한다.

### 4.2 Key Results (MVP)

| KR | 측정 기준 | 목표 |
|----|----------|------|
| KR1 | 투고된 작품 수 | 100개 |
| KR2 | 가입 크리에이터 수 | 50명 |
| KR3 | 추천을 통해 창작자 프로필을 조회한 횟수 | 30회 |
| KR4 | 추천→SNS 연락처 클릭 전환 | 10건 |

## 5. Market Segment

### 5.1 타겟 유저

**수익보다 협업 기회를 원하는 취미/인디 창작자**

| 특성 | 설명 |
|------|------|
| 동기 | 돈이 아니라 "같이 만드는 재미". 크레딧 표기만으로 충분 |
| 분야 | 음악(작곡/편곡/믹싱), 일러스트, 영상 편집, 3D 모델링, 성우, 사운드 디자인 등 |
| 현재 행동 | Twitter에서 "콜라보 구합니다" 트윗, DM으로 연락 |
| 불만 | 트윗이 묻힘, 역할별 검색 불가, 포트폴리오 확인 번거로움 |

### 5.2 초기 니치

**음악 MV 제작 커뮤니티** — 작곡가 + 일러스트레이터 + 영상 편집자의 협업 패턴이 가장 명확하고, 프로젝트 오너 본인이 첫 유저가 될 수 있는 분야.

### 5.3 제외

- 수익 목적의 프리랜서 (→ Fiverr, ココナラ 사용)
- 기업/상업 프로젝트 의뢰자
- 완성 에셋 구매자

## 6. Value Proposition

### 6.1 해결하는 문제 (Jobs to Be Done)

> "나는 곡을 만들었는데, 이 분위기에 맞는 일러스트레이터를 어디서 찾지?"

| 기존 방법 | 문제 |
|-----------|------|
| Twitter "콜라보 구합니다" 트윗 | 타임라인에 묻힘. 역할/장르 검색 불가 |
| Twitter DM으로 직접 연락 | 상대 포트폴리오 확인 번거로움. 매번 자기소개 반복 |
| pixiv/SoundCloud 검색 | 전시 목적. 협업 의사 확인 불가, 연락처 없음 |

### 6.2 Fugue가 제공하는 가치

| 가치 | 설명 |
|------|------|
| **30초 투고** | 링크 붙여넣기 + 태그. 기존 SNS 활동을 그대로 유지 |
| **작품 기반 추천** | "내 작품에 어울리는 다른 분야 작품"을 바로 발견 |
| **구조화된 탐색** | 분야별, 태그별 필터로 원하는 작품/창작자를 검색 |
| **즉시 연결** | 작품 → 창작자 프로필 → SNS 연락처 한 번에 도달 |

### 6.3 경쟁 대비 차별점

```
                  1:1 의뢰              작품 매칭
                    │                      │
  유료      VGen, Skeb, ココナラ            (없음)
                    │                      │
  무료      Piapro (투고만, 추천 없음)     ▶ Fugue ◀
```

### 7.1 User Flow

```
┌─────────────────────────────────────────────────────────┐
│                                                         │

│       │                                                 │
│       ▼                                                 │
│  [프로필 설정]                                            │

│       │                                                 │
│       ▼                                                 │
│  [작품 투고] ◄──────────────────────────────┐            │
```

### 7.2 Key Features

#### Feature 1: 작품 투고

작품을 직접 업로드하지 않고, 외부 플랫폼 링크를 붙여넣으면 자동으로 프리뷰가 생성된다.

**입력**
| 필드 | 필수 | 설명 |
|------|------|------|
| 작품 URL | O | SoundCloud, pixiv, YouTube, Twitter, 기타 URL |
| 분야 | O | 음악 / 일러스트 / 영상 / 3D / 사운드 / 기타 (URL에서 자동감지 시도, 수동 선택 가능) |
| 스타일 태그 | O | 최소 1개, 최대 5개. 프리셋 목록에서 선택 + 직접 입력 |
| 한 줄 설명 | X | 작품에 대한 짧은 설명 |

**동작**
- URL 붙여넣기 시 OG 메타데이터 fetch → 제목, 썸네일, 설명 자동 추출
- 지원 플랫폼(SoundCloud, YouTube, Twitter, pixiv)은 임베드 플레이어/프리뷰 표시
- 기타 URL은 OG 이미지 + 제목으로 카드형 프리뷰 표시
- 분야 자동감지: URL 도메인 기반 (soundcloud.com → 음악, pixiv.net → 일러스트 등)

**태그 프리셋 (초기)**

| 카테고리 | 태그 예시 |
|----------|----------|
| 분위기 | 감성, 밝은, 어두운, 몽환, 귀여운, 웅장한, 잔잔한 |
| 장르 (음악) | 일렉트로닉, 록, 팝, 클래식, 앰비언트, 힙합 |
| 장르 (비주얼) | 판타지, SF, 일상, 사이버펑크, 자연, 레트로 |
| 톤 | 밤, 낮, 도시, 바다, 우주, 숲 |

#### Feature 2: 크리에이터 프로필

| 필드 | 필수 | 설명 |
|------|------|------|
| 닉네임 | O | 표시명 |
| 역할 태그 | O | 복수 선택: 작곡, 일러스트, 영상편집, 3D모델링, 성우, 사운드디자인, 작사, 기타 |
| 한 줄 소개 | X | 자유 텍스트 |
| SNS 연락처 | O | 최소 1개: Twitter, Discord, 기타 (직접 입력) |
| 프로필 이미지 | X | 없으면 기본 아바타 |

- 투고한 작품 목록이 프로필 하단에 자동 표시 → 별도 포트폴리오 불필요
- 프로필 공개 여부는 기본 공개 (비공개 옵션 제공)

#### Feature 3: 추천 (작품 매칭)

핵심 기능. "내가 만들고 싶은 것"에 필요한 다른 분야의 작품을 추천한다.

**플로우**
1. 유저가 "뭘 만들고 싶으신가요?" 선택
2. 프로젝트 유형에 따라 필요한 분야가 자동 결정됨
3. 유저의 작품 태그와 같은 태그를 가진, 필요 분야의 작품을 추천

**프로젝트 유형별 필요 분야 템플릿**

| 프로젝트 유형 | 필요 분야 |
|--------------|----------|
| MV (뮤직비디오) | 일러스트, 영상 |
| 게임 | 일러스트, 음악, 사운드, 3D |
| 앨범 아트워크 | 일러스트 |
| 애니메이션 | 일러스트, 음악, 성우 |
| 보이스드라마 | 성우, 음악, 사운드 |
| 기타 | 유저가 필요 분야 직접 선택 |

**추천 로직 (v1)**
```
추천 작품 =
  WHERE 분야 IN (프로젝트 유형의 필요 분야)
  AND 태그 INTERSECT (내 작품 태그) >= 1개
  ORDER BY 태그 일치 수 DESC, 최신순
```

**추천 결과 표시**
- 필요 분야별 탭으로 구분 (예: MV → "일러스트" 탭, "영상" 탭)
- 각 탭에 카드 그리드: 썸네일 + 제목 + 태그 + 창작자명
- 카드 클릭 → 작품 상세 페이지

#### Feature 4: 탐색

추천과 별개로, 자유롭게 작품과 크리에이터를 검색하는 기능.

**작품 탐색**
- 필터: 분야, 태그 (복수 선택)
- 정렬: 최신순, 태그 일치순
- 결과: 카드 그리드 (추천 결과와 동일한 UI)

**크리에이터 탐색**
- 필터: 역할 태그
- 결과: 프로필 카드 (닉네임 + 역할 + 한 줄 소개 + 대표 작품 1개)

#### Feature 5: 인증

MVP에서는 최소한의 인증만 구현.

| 방법 | 설명 |
|------|------|
| 소셜 로그인 | Twitter OAuth 또는 Discord OAuth (타겟 유저의 주 SNS) |
| 이메일/비밀번호 | 소셜 로그인 외 대안 |

### 7.3 Assumptions

| 가정 | 리스크 | 검증 방법 |
|------|--------|----------|
| 링크 투고만으로 충분한 작품이 모인다 | Cold Start | 투고 허들 최소화 + 음악 MV 니치부터 시작 |
| 태그 매칭으로 "어울리는" 작품을 찾을 수 있다 | 추천 품질 | v1 출시 후 추천→프로필 조회 전환율 측정 |
| 추천받은 작품의 창작자에게 실제로 연락한다 | 협업 전환 | SNS 연락처 클릭 수 추적 |
| 창작자가 "크레딧 표기" 조건에 동의하고 협업한다 | 라이선스 수용 | 초기 유저 피드백 |

### 8.2 Post-MVP (검증 후)

| 기능 | 트리거 |
|------|--------|
| 프로젝트 보드 (팀 결성 기능) | 추천→협업 전환이 검증되면 |
| 추천 고도화 (협업 이력/인기도 가중치) | 데이터 충분히 축적되면 |
| 임베딩 기반 cross-domain 추천 | 태그 매칭의 한계가 명확해지면 |
| 알림 (새 작품, 태그 매칭) | 유저 리텐션 개선 필요 시 |
| 에셋 마켓플레이스 (유료 거래) | 유저 수요 확인 후 |
| 인앱 메시징 | 외부 SNS 연락의 불편이 증명되면 |
