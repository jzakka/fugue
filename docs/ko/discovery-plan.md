# Discovery Plan: Fugue — 창작물 기반 협업 매칭 플랫폼

**Date**: 2026-03-29
**Product Stage**: 신규 (토이 프로젝트)
**Discovery Question**: 취미 창작자들이 자신의 작품을 기반으로 협업에 필요한 다른 작품을 쉽게 발견할 수 있는 MVP는?

---

## 배경

- 개인 작곡 작품을 YouTube MV로 만들고 싶지만 일러스트/영상 편집 역량이 없어서 시작된 아이디어
- 레퍼런스: piapro (창작물 투고 + 라이선스 공유 사이트)
- 현재 시장: Twitter DM을 통한 비구조적 1:1 매칭이 주류

## 타겟 유저

**수익보다 협업 기회를 원하는 취미/인디 창작자**

- 인디 작곡가, 취미 일러스트레이터, 영상 편집자 등
- 수익 목적 유저는 초기 타겟에서 제외
- 라이선스: "크레딧 표기 시 자유 사용" 기본값

## 시장 조사

### 포지셔닝 맵

```
                  1:1 의뢰              팀/작품 매칭
                    │                      │
  유료      VGen, Skeb, ココナラ            (없음)
            Fiverr, Ko-fi
                    │                      │
  무료      Piapro (투고+2차창작)          ▶ Fugue ◀
```

### 경쟁 서비스

| 서비스 | 모델 | Fugue와의 차이 |
|--------|------|----------------------|
| Piapro | 작품 투고 + 라이선스 2차 창작 | 특정 생태계 한정. 투고만 있고 매칭/추천 없음 |
| VGen | 유료 아트 커미션 (5%) | 커미션 거래 중심. 무료 협업 아님 |
| Skeb | 원웨이 유료 커미션 | 협상 없는 의뢰 모델. 협업 아님 |
| ココナラ | 스킬 마켓플레이스 (22%) | 범용 프리랜서 거래. 창작 협업 특화 아님 |
| Fiverr | 범용 프리랜서 마켓 (20%) | 거래 중심. 취미 창작자 타겟 아님 |
| Ko-fi | 후원 + 커미션 | 팬→크리에이터 후원 모델 |
| ArtStation/Behance | 포트폴리오 전시 | 전시만 있고 협업 매칭 없음 |
| BOOTH | 에셋/굿즈 판매 | 판매 중심. pixiv 생태계 |
| nizima | Live2D 모델 마켓 | VTuber 특화 에셋 거래 |

### 핵심 인사이트

- **"무료 + 작품 기반 협업 매칭"은 아무도 안 하고 있음**
- Piapro도 투고 + 라이선스 공유이지 매칭/추천은 아님
- 기존 서비스는 전부 "거래(커미션)" 중심
- 취미 창작자의 무료 협업은 여전히 Twitter DM에 의존

## 핵심 컨셉

**사람이 아니라 "작품"을 매칭한다.**

"일러스트레이터 구합니다"가 아니라 "내 음악에 어울리는 일러스트를 찾고 싶다"

### 핵심 플로우

```
1. 투고 (30초)
   작품 링크 붙여넣기 (SoundCloud/pixiv/YouTube/Twitter)
   → OG 메타데이터로 자동 프리뷰
   → 분야 자동감지 + 태그 선택

2. "뭘 만들고 싶으신가요?"
   프로젝트 유형 선택 (MV / 게임 / 앨범아트 / 애니메이션 등)
   → 내 작품 선택

3. 추천
   프로젝트 유형에 필요한 다른 분야 작품을 태그 기반으로 추천
   → 작품 상세에서 창작자 프로필/SNS로 직접 연결
```

## MVP 스코프

### 포함

#### 1. 작품 투고 (외부 링크 기반)
- 외부 링크 붙여넣기 → 임베드/OG 프리뷰 자동 생성
- 지원 플랫폼: SoundCloud, pixiv, YouTube, Twitter, 기타 URL
- 분야 태그: 음악, 일러스트, 영상, 3D, 사운드 등
- 스타일/분위기 태그: 감성, 밤, 일렉트로닉, 판타지 등
- 라이선스: "크레딧 표기 시 자유 사용" 기본값

#### 2. 크리에이터 프로필
- 역할 태그 (복수 선택)
- 한 줄 소개
- SNS 연락처 (Twitter/Discord/기타)
- 투고한 작품 목록이 자동으로 포트폴리오가 됨

#### 3. 추천 (작품 매칭)
- "뭘 만들고 싶으신가요?" → 프로젝트 유형 선택
- 프로젝트 유형별 필요 분야 템플릿 (MV → 일러스트 + 영상편집)
- 같은 태그를 가진 다른 분야 작품 추천
- 작품 상세 → 창작자 프로필 → SNS 연락처로 직접 연결

#### 4. 탐색
- 분야별, 태그별 필터
- 작품 검색 / 크리에이터 검색

### 제외 (검증 후 추가)

| 기능 | 제외 이유 | 대체 수단 |
|------|----------|----------|
| 프로젝트 보드/허브 | 투고+추천 검증이 우선 | 추천→연락→외부에서 협업 |
| AI 기반 추천 | 초기 데이터 부족 | 태그 매칭 + 유형별 템플릿 |
| 결제/정산 | 무료 협업 타겟 | 제거 |
| 멀티미디어 저장 | 인프라 비용 | 외부 링크 임베드로 대체 |
| DM/채팅 | 오버엔지니어링 | Twitter DM/Discord |

### 추천 로직 로드맵

| 단계 | 방법 | 시기 |
|------|------|------|
| v1 | 태그 매칭 + 프로젝트 유형 템플릿 | MVP |
| v2 | 협업 이력/인기도 가중치 | 데이터 축적 후 |
| v3 | 임베딩 기반 cross-domain 유사도 | 장기 |

## Cold Start 전략

1. **투고 허들 최소화** — 링크 붙여넣기 + 태그 선택, 30초면 끝
2. **한쪽(공급)만 먼저 모으기** — 작품 투고만으로도 가치 있음 (포트폴리오 + 노출 기회)
3. **본인이 첫 번째 유저** — 작곡 작품 투고 후 "MV 만들기" 추천 플로우 직접 테스트
4. **니치 커뮤니티 타겟** — 음악 MV 커뮤니티부터 시작
5. **기존 SNS 활동 유지** — 외부 링크 기반이므로 전환 비용 0

## 핵심 리스크

| 리스크 | 설명 | 완화 전략 |
|--------|------|----------|
| 전환 동기 부족 | "왜 여기에 링크를 올려야 해?" | 추천을 통한 협업 기회 = 기존 SNS에 없는 고유 가치 |
| Cold Start | 작품이 부족하면 추천이 안 됨 | 투고 허들 최소화 + 한쪽만 먼저 모으기 |
| 태그 매칭 품질 | 태그만으로 "어울리는" 작품을 찾을 수 있는가 | v1에서 검증 후 추천 고도화 |
| 추천→협업 전환 | 추천받아도 실제 연락으로 이어지는가 | 작품 상세에서 창작자 SNS 직접 연결 |
