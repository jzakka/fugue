# Discovery Plan: Fugue — 크로스미디어 창작물 큐레이션 플랫폼

**Date**: 2026-04-03
**Product Stage**: 신규 (사이드 프로젝트 / 포트폴리오)
**Discovery Question**: 분야에 관계없이 웹에 흩어진 창작물을 한곳에서 발견하고 큐레이션할 수 있는 MVP는?

---

## 배경

- 창작물은 플랫폼별로 흩어져 있음: 음악은 SoundCloud, 일러스트는 pixiv, 영상은 YouTube, 코드는 GitHub
- 각 플랫폼은 자기 분야만 다루기 때문에 크로스 분야 탐색이 불가능
- "이 음악에 어울리는 일러스트"를 찾으려면 여러 플랫폼을 돌아다녀야 함
- 기존 큐레이션 서비스 (Pinterest, Are.na)는 이미지에 특화. 음악/코드는 2등 시민 취급

## 타겟 유저

**다양한 분야의 창작물을 탐색하고 모으는 것을 즐기는 사람**

- 영감을 위해 수집하는 크리에이터, 레퍼런스 정리하는 디자이너, 취향 큐레이션을 즐기는 사람
- 분야 무관: 음악, 일러스트, 영상, 글, 코드, 3D, 게임, 사운드 등
- 초기 니치: 음악 MV 제작에 관심있는 크리에이터 (크로스 분야 탐색 패턴이 가장 명확)

## 시장 조사

### 포지셔닝 맵

```
              이미지만           크로스미디어
                │                    │
  추천 있음   Pinterest             ▶ Fugue ◀
                │                    │
  추천 없음   Are.na, Raindrop     (없음)
```

### 참고 서비스

| 서비스 | 모델 | Fugue와의 차이 |
|--------|------|---------------|
| Pinterest | 이미지 큐레이션 (핀 + 보드 + 추천) | 이미지에 특화. 음악/코드 핀 불가 |
| Are.na | 리서치/아카이빙 큐레이션 | 추천 없음. 니치 커뮤니티 |
| Raindrop.io | 북마크 매니저 | 정리 도구. 큐레이션/추천 아님 |
| Piapro | 창작물 투고 + 라이선스 공유 | 보컬로이드 생태계 한정. 추천 없음 |
| SoundCloud/pixiv/YouTube | 단일 분야 플랫폼 | 자기 분야만. 크로스 탐색 불가 |

### 핵심 인사이트

- **크로스미디어 큐레이션 + 추천은 아무도 안 하고 있음**
- Pinterest가 이미지에서 증명한 모델 (핀 + 보드 + 추천)을 모든 창작 미디어로 확장
- 각 플랫폼에 갇힌 창작물을 분야를 넘나들며 발견하는 것이 핵심 가치

## 핵심 컨셉

**Pinterest for all creative media.**

음악, 일러스트, 영상, 글, 코드가 하나의 피드에서 공존한다. 클릭할수록 내 취향을 학습해서 더 좋은 작품이 뜬다.

### 핵심 플로우

```
1. 핀 (30초)
   URL 붙여넣기 (SoundCloud/pixiv/YouTube/GitHub/어디든)
   → OG 메타데이터로 자동 프리뷰
   → 분야 자동감지 + 태그 선택

2. 보드
   핀을 주제별로 묶어 컬렉션 구성
   → "몽환적인 인디 음악", "사이버펑크 일러스트", "크리에이티브 코딩" 등

3. 발견
   추천 기반 피드에서 새로운 작품 탐색
   → 클릭/핀 행동이 취향으로 반영됨
   → 작품 상세에서 연관 작품 발견
```

## MVP 스코프

### 포함

#### 1. 핀 (Pin)
- 외부 URL 큐레이션. 소유권 주장이 아닌 큐레이션.
- OG 자동 프리뷰 + 도메인 기반 분야 자동감지
- 스타일 태그 (1~5개)
- Create, Read, Delete만 (Update 없음)

#### 2. 보드 (Board)
- 핀을 주제별로 묶는 컬렉션
- 공개/비공개 설정
- 한 핀이 여러 보드에 속할 수 있음

#### 3. 추천 기반 피드
- 태그 빈도 기반 휴리스틱 추천 (v1)
- 콜드 스타트: 핀 10개 미만이면 최신순
- 배치/Redis 캐싱

#### 4. 암묵적 취향 학습
- 조회/핀/보드 추가 이벤트 기록
- 추천 엔진의 입력 데이터

#### 5. 연관 작품
- 작품 상세 하단에 태그 유사 작품 표시

#### 6. 인증 (구현 완료)
- Google OAuth, Discord OAuth

### 제외 (검증 후 추가)

| 기능 | 제외 이유 | 대체 수단 |
|------|----------|----------|
| 협업 매칭 | 큐레이션에 집중 | 플랫폼 외부에서 자연스럽게 |
| ML 기반 추천 | 초기 데이터 부족 | 태그 휴리스틱으로 시작 |
| 임베드 플레이어 | OG 프리뷰로 충분 | 외부 링크로 이동 |
| DM/채팅 | 오버엔지니어링 | 외부 SNS |
| 파일 호스팅 | 인프라 비용 | 외부 링크 기반 |

### 추천 로직 로드맵

| 단계 | 방법 | 시기 |
|------|------|------|
| v1 | 태그 빈도 휴리스틱 | MVP |
| v2 | 피처스토어 도입 | 데이터 축적 후 |
| v3 | ML 모델 학습 | 피처스토어 기반 |

## Cold Start 전략

1. **핀 허들 최소화** — URL 붙여넣기 + 태그, 30초면 끝
2. **본인이 첫 유저** — 직접 좋아하는 작품 핀하면서 피드 채우기
3. **니치 커뮤니티** — 음악 MV 크리에이터부터 시작
4. **기존 SNS 활동 유지** — 외부 링크 기반이므로 전환 비용 0
5. **보드 공유** — SNS에 보드 링크 공유로 유입

## 핵심 리스크

| 리스크 | 설명 | 완화 전략 |
|--------|------|----------|
| 전환 동기 부족 | "왜 여기에 링크를 올려야 해?" | 보드 정리 + 추천 피드 = 브라우저 북마크보다 나은 경험 |
| Cold Start | 작품이 부족하면 추천이 안 됨 | 핀 허들 최소화 + 본인이 첫 유저 |
| 태그 매칭 품질 | 태그만으로 취향 매칭이 되는가 | v1에서 검증 후 피처스토어/ML로 진화 |
| OG fetch 제한 | 일부 플랫폼의 OG 메타데이터 부실 | 수동 입력 폴백 제공 |
