## 技术栈

### 应用层

| 类别 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js | React框架，SSR/SSG支持 |
| 后端API | Go + Chi | 轻量级HTTP路由器 |
| SQL生成 | sqlc | 类型安全的SQL → Go代码生成 |
| 数据库 | PostgreSQL | 关系型数据库 |
| 缓存 | Redis | 会话管理、缓存 |
| 认证 | OAuth 2.0 | Google、GitHub社交登录 |

### 基础设施 / Terraform

| 类别 | 技术 | 说明 |
|------|------|------|
| 网络 | VPC | 10.0.0.0/16，3层子网 |
| 容器编排 | EKS Spot | Kubernetes托管服务，Spot实例降低成本 |
| 镜像仓库 | ECR | 容器镜像私有仓库 |
| 负载均衡 | ALB Controller | Kubernetes Ingress → ALB自动创建 |
| DNS | Route 53 | 域名管理 |
| SSL | ACM | 免费SSL证书，自动续期 |
| 权限管理 | IRSA | EKS Pod级别的IAM角色绑定 |

### CI/CD

| 类别 | 技术 | 说明 |
|------|------|------|
| CI | GitHub Actions | 测试、构建、镜像推送 |
| CD | ArgoCD | GitOps方式的K8s部署 |
| 基础设施即代码 | Terraform | AWS资源声明式管理 |
| 包管理 | Helm | Kubernetes应用包管理 |

### 可观测性

| 类别 | 技术 | 说明 |
|------|------|------|
| 指标 | Prometheus + Grafana | 指标收集与可视化 |
| 日志 | Loki + Promtail | 日志聚合与查询 |
| 链路追踪 | Tempo + OpenTelemetry | 分布式链路追踪 |
| 告警 | Grafana Alerting | 基于指标/日志的告警 |

## 网络设计

### VPC 结构

```
VPC: 10.0.0.0/16
│
├── 公有子网
│   ├── 10.0.1.0/24 (AZ-a)
│   └── 10.0.2.0/24 (AZ-c)
│
├── 私有子网
│   ├── 10.0.3.0/24 (AZ-a)
│   └── 10.0.4.0/24 (AZ-c)
│
└── 数据库子网
    ├── 10.0.5.0/24 (AZ-a)
    └── 10.0.6.0/24 (AZ-c)
```

### 子网设计

| 子网类型 | CIDR | 可用区 | 用途 | 对外访问 |
|----------|------|--------|------|----------|
| 公有子网 | 10.0.1.0/24 | AZ-a | ALB、NAT | 直连互联网 |
| 公有子网 | 10.0.2.0/24 | AZ-c | ALB（多AZ） | 直连互联网 |
| 私有子网 | 10.0.3.0/24 | AZ-a | EKS节点 | 通过NAT |
| 私有子网 | 10.0.4.0/24 | AZ-c | EKS节点 | 通过NAT |
| 数据库子网 | 10.0.5.0/24 | AZ-a | RDS、ElastiCache | 不可访问 |
| 数据库子网 | 10.0.6.0/24 | AZ-c | RDS、ElastiCache | 不可访问 |

### 流量路径

#### 入站流量

```
用户 → Route 53 → ALB (公有子网) → EKS Pod (私有子网)
```

#### 内部流量

```
EKS Pod (私有子网) → RDS/ElastiCache (数据库子网)
```

#### 出站流量

```
EKS Pod (私有子网) → fck-nat (公有子网) → 互联网
```

### 安全组链

```
┌──────────┐     ┌──────────┐     ┌──────────────┐
│  ALB SG  │────▶│  EKS SG  │────▶│    DB SG     │
│          │     │          │     │              │
│ 入站:    │     │ 入站:    │     │ 入站:        │
│ 80/443   │     │ 来自     │     │ 来自         │
│ (全部)   │     │ ALB SG   │     │ EKS SG       │
└──────────┘     └──────────┘     └──────────────┘
```

| 安全组 | 入站规则 | 说明 |
|--------|----------|------|
| ALB SG | 80/443 来自 0.0.0.0/0 | 互联网流量接收 |
| EKS SG | 全端口来自 ALB SG | 仅允许来自ALB的流量 |
| DB SG | 5432/6379 来自 EKS SG | 仅允许来自EKS的数据库/缓存连接 |

### CI 流水线详情 (GitHub Actions)

```yaml
# .github/workflows/ci.yml (概念示意)
触发条件: push 至 main / PR

步骤:
  1. 测试
     - Go: go test ./...
     - Web: npm test
     - Lint: golangci-lint, eslint

  2. 构建
     - Go API: Docker build
     - Next.js: Docker build

  3. 推送
     - ECR 登录
     - 镜像 tag: git SHA
     - 推送至 ECR

  4. 更新清单
     - 更新 Helm values 中的镜像 tag
     - 提交变更至 Git仓库
```

### CD 流水线详情 (ArgoCD)

| 环境 | 同步策略 | 说明 |
|------|----------|------|
| dev | 自动同步 | Git仓库中Helm values变更 → 自动部署 |
| prod | 手动同步 | 在dev验证后 → 手动批准 → 部署至生产 |

## 月度成本估算

### 总成本: 约 $159/月

| 项目 | 服务 | 规格 | 月费用 |
|------|------|------|--------|
| EKS 控制面 | EKS | 托管控制面 | $74 |
| 计算 (app) | EC2 Spot | t3.medium × 2 | ~$30 |
| 计算 (monitoring) | EC2 Spot | t3.large × 1 | ~$19 |
| NAT | fck-nat | t4g.nano | ~$3 |
| 数据库 | RDS | db.t3.micro PostgreSQL | ~$15 |
| 缓存 | ElastiCache | cache.t3.micro Redis | ~$12 |
| DNS | Route 53 | 托管区域 + 查询 | ~$1 |
| 其他 | S3, ECR, CloudWatch等 | - | ~$5 |

### 成本优化策略

| 策略 | 说明 | 预计节省 |
|------|------|----------|
| Spot实例 | EKS节点使用Spot实例 | 比按需实例节省约60-70% |
| fck-nat | 使用开源NAT替代NAT Gateway | $30+/月 → ~$3/月 |
| dev环境内部DB | dev环境使用CloudNativePG/Redis（K8s内） | 无额外RDS/ElastiCache费用 |
| 单NAT | 使用单个NAT实例（非高可用） | 仅1个实例费用 |
| t3.micro RDS | 最小规格RDS（初期足够） | 最低成本起步 |

### 扩展时的成本变化

| 阶段 | 预计变更 | 追加费用 |
|------|----------|----------|
| 用户增加 (100+) | app节点组扩展至3-4台 | +$15~30/月 |
| 数据增长 | RDS存储扩展 | +$5~10/月 |
| 高可用需求 | 多AZ RDS、NAT Gateway | +$50~100/月 |
| AI推荐 (v3) | GPU实例或外部API | +$50~200/月 |

## 策展模型的基础设施影响

### 新组件与现有基础设施映射

| 组件 | 基础设施位置 | 变更事项 |
|------|------------|----------|
| OG fetch服务 | Go API Pod内部 | 出站流量增加（经fck-nat） |
| interactions表 | RDS (prod) / CNPG (dev) | 高写入表。每次页面查看都INSERT |
| 推荐缓存 | ElastiCache Redis (prod) | 按用户缓存推荐结果（TTL 5分钟） |
| boards/board_pins | RDS (prod) / CNPG (dev) | 普通CRUD。无需基础设施变更 |

### 推荐引擎基础设施演进路线

- **v1 (MVP)**: Go API内部查询 + Redis缓存。无需额外基础设施
- **v2**: K8s CronJob定期批量计算。在Helm template添加CronJob
- **v3**: ML训练管道。SageMaker或GPU节点

### MVP阶段的变更

VPC、EKS、CI/CD、Security Group、监控、成本: **全部无变更**
