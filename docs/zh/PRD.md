# PRD: Fugue

---

## 1. 概要

**Fugue**是一个面向业余/独立创作者的协作匹配平台，创作者可以发布作品链接，系统会推荐互补的作品，从而促进协作。

### 核心原则

- **免费协作**: 以"标注出处即可免费使用"为默认授权
- **匹配作品，而非匹配人**: 不是"寻找插画师"，而是"寻找与我音乐相配的插画"
- **极简发布**: 30秒内完成作品发布

### 吉祥物

Fugue的吉祥物是一只**戴着耳机、手持画笔的河豚（fugu/河豚）**——Fugue与Fugu的文字游戏，象征音乐与视觉创作的融合。

## 3. 背景

### 3.1 背景情境

> 我为自己作的曲子想制作YouTube MV，但我不会画插画，也不会做视频编辑。
> 在Twitter上搜索，虽然有很多优秀的插画师和视频编辑者，但没有系统化的方式来找到"与我音乐氛围相配的作品"。

- 作为业余作曲者，有创作完成的曲子
- 想为曲子制作MV，但缺乏插画/视频编辑能力
- 现有的方式是通过Twitter DM逐一联系，效率低且非结构化

### 3.2 为什么是现在？

| 理由 | 说明 |
|------|------|
| 独立创作者市场增长 | YouTube、SoundCloud等平台降低了创作门槛，独立创作者数量持续增加 |
| 现有平台皆为收费委托模式 | VGen、Skeb、Coconala、Fiverr等都是收费委托平台，不适合"免费协作"场景 |
| Piapro局限于特定生态系统 | 唯一的免费协作参考案例，但仅限Vocaloid/Crypton生态系统 |
| 跨领域协作需求增加 | MV制作、专辑封面等跨领域协作需求日益增长 |

### 3.3 参考服务

| 服务 | 参考点 | 局限性 |
|------|--------|--------|
| Piapro | 免费协作模式，作品发布与浏览 | 仅限Vocaloid/Crypton生态系统 |
| VGen | 创作者匹配流程 | 收费委托模式，非免费协作 |
| Skeb | 简洁的委托UX | 收费模式，匿名，沟通受限 |
| Coconala | 技能市场结构 | 收费模式，以专业服务为主 |

## 4. 目标

### 目标声明

> 帮助业余创作者轻松发现与自己作品互补的作品和创作者。

### 关键结果（KR）

| KR | 目标值 | 衡量方式 |
|----|--------|----------|
| 发布的作品数 | 100件 | 数据库作品记录数 |
| 注册创作者数 | 50人 | 注册用户数 |
| 通过推荐查看资料的次数 | 30次 | 从推荐结果点击进入创作者资料页 |
| SNS联系点击数 | 10次 | 点击创作者资料中的SNS链接 |

## 5. 市场细分

### 目标市场

**比起金钱收益更渴望协作机会的业余/独立创作者**

### 初始利基市场

**音乐MV制作社区**

- 独立作曲者 + 业余插画师 + 视频编辑者
- 围绕"为一首歌制作MV"这一具体目标形成协作

### 目标用户画像

| 画像 | 特征 |
|------|------|
| 独立作曲者 | 在SoundCloud/YouTube上发布自己的曲子，想制作MV但缺乏视觉创作能力 |
| 业余插画师 | 在pixiv/Twitter上发布作品，想让插画被用于更大的项目中 |
| 视频编辑者 | 有剪辑技术但缺少素材，想找音乐和插画来制作MV |

### 排除的用户

| 类型 | 排除理由 |
|------|----------|
| 专业自由职业者 | 需要收费功能，与"免费协作"核心价值冲突 |
| 企业客户 | B2B需求超出个人项目范围 |
| 数字素材购买者 | 购买/销售模式，非协作模式 |

### 7.1 用户流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   社交登录    │────▶│  创建资料    │────▶│  发布作品    │
│ (Google/     │     │  昵称/角色/  │     │  粘贴链接/   │
│  GitHub)     │     │  简介/SNS   │     │  标签/领域   │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                                │
                    ┌─────────────┐     ┌───────▼──────┐
                    │  查看推荐    │◀────│ 选择项目类型  │
                    │  匹配的作品  │     │  (例: 音乐MV) │
                    └──────┬──────┘     └──────────────┘
                           │
                    ┌──────▼──────┐
                    │  查看创作者  │
                    │  资料/SNS   │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ 通过SNS联系  │
```

### 7.2 功能详述

#### 功能1: 作品发布

| 项目 | 说明 |
|------|------|
| 概述 | 粘贴外部链接发布作品 |
| 输入 | URL、领域、标签、描述（可选） |
| 处理 | OGP/oEmbed解析自动生成预览 |
| 输出 | 带预览的作品卡片 |
| 优先级 | P0 |

**支持的URL类型:**

| 平台 | URL格式 | 预览方式 |
|------|---------|----------|
| YouTube | youtube.com/watch?v=xxx | oEmbed嵌入播放器 |
| SoundCloud | soundcloud.com/xxx/xxx | oEmbed嵌入播放器 |
| pixiv | pixiv.net/artworks/xxx | OGP缩略图 |
| Twitter/X | x.com/xxx/status/xxx | OGP缩略图 |
| niconico | nicovideo.jp/watch/xxx | oEmbed嵌入播放器 |
| 其他 | 任意URL | OGP缩略图（若有） |

#### 功能2: 创作者资料

| 项目 | 说明 |
|------|------|
| 概述 | 创作者的个人信息和作品列表页面 |
| 字段 | 昵称、角色（多选）、简介、SNS链接（多个）、头像 |
| 关联 | 该创作者发布的所有作品列表 |
| 优先级 | P0 |

**角色选项:**

| 角色 | 说明 |
|------|------|
| 作曲 | 音乐创作 |
| 编曲 | 音乐编曲 |
| 作词 | 歌词创作 |
| 演唱 | 歌手/声优 |
| 插画 | 插画/绘画 |
| 视频 | 视频编辑/动态图形 |
| 设计 | 平面设计/排版 |
| 3D | 3D建模/动画 |
| 其他 | 上述类别以外的角色 |

#### 功能3: 推荐/作品匹配

| 项目 | 说明 |
|------|------|
| 概述 | 基于项目类型和标签推荐互补的作品 |
| 输入 | 用户的作品 + 选择的项目类型 |
| 逻辑 | 1) 确定项目类型所需的领域 2) 在该领域中搜索标签匹配的作品 3) 按匹配度排序 |
| 输出 | 推荐作品列表（带匹配度） |
| 优先级 | P0 |

**项目类型模板:**

| 项目类型 | 所需领域 | 说明 |
|----------|----------|------|
| 音乐MV | 插画, 视频 | 为一首歌制作音乐视频 |
| 专辑封面 | 插画, 设计 | 为专辑/单曲制作封面图 |
| 歌曲插画 | 插画 | 为一首歌配上一幅插画 |
| 原创动画 | 插画, 视频, 音乐 | 制作原创动画短片 |
| 游戏原声 | 作曲, 编曲 | 为独立游戏制作音乐 |
| VOCALOID MV | 插画, 视频 | 为Vocaloid歌曲制作MV |
| 自定义 | 用户自选 | 用户自由选择所需领域 |

**标签预设:**

| 类别 | 标签示例 |
|------|----------|
| 氛围 | 柔和、激烈、忧郁、明快、梦幻、黑暗、温暖、清凉 |
| 音乐流派 | 流行、摇滚、电子、古典、爵士、嘻哈、民谣、氛围音乐 |
| 插画风格 | 动漫、写实、水彩、像素画、扁平、简约、复古、赛博朋克 |
| 视频风格 | 动态图形、实拍、动画、拼贴、故障效果、循环、电影感 |
| 季节/时间 | 春、夏、秋、冬、白天、夜晚、黄昏、黎明 |
| 色调 | 暖色、冷色、单色、彩色、粉彩、霓虹 |

**匹配逻辑（v1）:**

```
输入:
  - 用户作品标签: [柔和, 钢琴, 抒情, 冬天]
  - 项目类型: 音乐MV
  - 所需领域: [插画, 视频]

处理:
  1. 在"插画"和"视频"领域中搜索作品
  2. 计算标签交集
     候选A标签: [柔和, 水彩, 抒情, 雪]    → 交集: [柔和, 抒情] → 分数: 2
     候选B标签: [激烈, 像素画, 摇滚]       → 交集: []           → 分数: 0
     候选C标签: [柔和, 动画, 冬天, 梦幻]   → 交集: [柔和, 冬天] → 分数: 2
  3. 按分数降序排序

输出:
  1. 候选A (插画) - 匹配度: 50%
  2. 候选C (视频) - 匹配度: 50%
```

#### 功能4: 探索

| 项目 | 说明 |
|------|------|
| 概述 | 按各种条件浏览和筛选作品 |
| 筛选条件 | 领域、标签、项目类型 |
| 排序 | 最新、热门（查看数）、推荐 |
| 优先级 | P0 |

#### 功能5: 认证

| 项目 | 说明 |
|------|------|
| 概述 | 社交登录，无需邮箱注册 |
| 支持的提供商 | Google, GitHub |
| 流程 | OAuth 2.0 → 创建用户 → 创建资料 |
| 优先级 | P0 |

### 7.3 前提假设

| 假设 | 验证方法 | 风险 |
|------|----------|------|
| 业余创作者愿意免费分享作品用于协作 | 发布作品数量 | 可能创作者更倾向有偿方式 |
| 标签匹配能提供有意义的推荐 | 通过推荐查看资料的比例 | 标签可能不够精准 |
| 30秒发布足以降低进入门槛 | 发布完成率 | 用户可能还是觉得麻烦 |
| SNS联系方式足以支撑初期协作 | SNS点击数 | 用户可能需要站内沟通功能 |
| 音乐MV社区是合适的初始利基市场 | 该社区用户占比 | 可能市场太小 |

## 8. 发布计划

### MVP（所有P0功能）

| 功能 | 状态 | 说明 |
|------|------|------|
| 作品发布 | P0 | 外部链接 + 自动预览 + 标签 |
| 创作者资料 | P0 | 昵称/角色/简介/SNS/头像 |
| 推荐/作品匹配 | P0 | 项目类型模板 + 标签匹配 |
| 探索 | P0 | 领域/标签/项目类型筛选 |
| 认证 | P0 | Google/GitHub社交登录 |

### Post-MVP功能

| 功能 | 触发条件 | 优先级 |
|------|----------|--------|
| 项目看板 | 协作管理需求出现时（用户反馈） | P1 |
| 站内消息 | 用户反馈SNS联系方式不够用时 | P1 |
| AI推荐（v2/v3） | 标签匹配的推荐质量不够好时 | P2 |
| 媒体直接上传 | 外部链接方式不满足需求时 | P2 |
| 支付功能 | 出现付费委托需求时 | P3 |
| 多语言支持 | 海外用户增长时 | P2 |
| 移动端App | Web端用户量足够大时 | P3 |
| 协作历史/作品集 | 完成的协作项目累积后 | P2 |
| 通知系统 | 推荐/匹配频繁发生时 | P1 |
| 收藏/点赞 | 用户探索行为增加时 | P1 |
