## 1. 概要

Fugue是一个跨媒体创作策展平台。Pinterest仅限于图片，而Fugue可以跨领域发现和策展音乐、插画、视频、文字、代码、3D、游戏等所有类型的创作内容。

- 将外部创作URL"钉"住 → 通过OG元数据自动生成预览
- 将钉子整理到主题看板中
- 基于点击/钉子行为学习品味，提供个性化信息流
- 作品详情页自动展示相关作品

## 3. 背景

### 3.1 背景

创作内容分散在各个平台上：音乐在SoundCloud，插画在pixiv，视频在YouTube，代码在GitHub。每个平台只处理自己的领域，无法发现"与这首音乐搭配的插画"或"适合这个游戏的BGM"。

现有的策展服务（Pinterest、Are.na等）主要专注于图片，无法平等地处理音乐或代码等媒体。

### 3.2 为什么是现在？

- 独立/业余创作市场持续增长（VTuber、独立游戏、同人音乐等）
- 创作内容分散的平台不断增多
- Pinterest以图片为中心。Are.na是小众。跨媒体策展是空白领域

### 3.3 参考

- **Pinterest**: 图片策展的教科书。钉子+看板+推荐信息流模式
- **Are.na**: 研究/存档目的的策展
- **Piapro**: 创作投稿+许可共享。限于Vocaloid生态系统
- **Raindrop.io**: 书签管理器。更偏向整理工具而非策展

| 特征 | 说明 |

---

## 6. 解决方案

### Feature 1: 钉子 (Pin)
外部创作URL的策展。OG自动预览。基于域名的领域自动检测。

### Feature 2: 看板 (Board)
将钉子按主题分组的集合。公开/私密设置。

### Feature 3: 推荐信息流
基于用户品味的作品推荐。v1: 标签频率启发式。路线图: 启发式 → 特征存储 → ML。

### Feature 4: 隐式品味学习
将view、pin、board_add事件记录到interactions表中。

### Feature 5: 相关作品
在作品详情页底部基于标签相似度展示相关作品。最多10件。

### Feature 6: 认证（已实现）
Google OAuth、Discord OAuth。基于JWT的认证。
