## 1. 背景

> 我为自己作的曲子想制作YouTube MV，但我不会画插画，也不会做视频编辑。

### 个人经历

- 作为业余作曲者，有好几首完成的曲子
- 想为这些曲子制作MV，但缺乏插画和视频编辑能力
- 在Twitter上通过DM寻找合作者，但流程非结构化，匹配效率低

### 参考案例

- **piapro**: Crypton Future Media运营的创作者协作平台
  - 以Vocaloid创作者为中心的生态系统
  - 可以上传音乐、插画、歌词等作品并互相授权使用
  - 局限性：仅限于特定生态系统（Vocaloid/Crypton）

### 现状问题

| 问题 | 说明 |
|------|------|
| 非结构化匹配 | 通过Twitter DM进行1:1联系，没有系统化的匹配流程 |
| 信息不对称 | 难以判断对方的作品风格和水平 |
| 进入壁垒 | 对于初次合作的创作者来说，联系陌生人的心理负担很大 |
| 发现困难 | 难以找到与自己作品氛围相匹配的合作者 |

## 2. 目标用户

### 核心目标用户

**比起收益更渴望合作机会的业余/独立创作者**

| 角色 | 特征 | 需求 |
|------|------|------|
| 独立作曲者 | 有完成的曲子，想制作MV | 寻找与音乐氛围相配的插画/视频 |
| 业余插画师 | 作为爱好进行创作，想让作品被更多人看到 | 希望自己的插画被用于音乐/视频等项目 |
| 视频编辑者 | 具备剪辑技术，寻找可用素材 | 想找到音乐和插画来制作MV |

### 用户特征

- **动机**: 创作的乐趣 > 金钱收益
- **协作方式**: 免费使用，标注出处作为默认规则
- **活动平台**: Twitter、YouTube、SoundCloud、pixiv等
- **授权模式**: "标注出处即可免费使用"作为默认授权

### 核心洞察

> 目前没有人在做"免费 + 基于作品的协作匹配"。

- 现有平台要么是**收费委托**（VGen、Skeb、Fiverr）
- 要么是**单纯的作品展示**（ArtStation、Behance）
- Piapro最接近，但**局限于特定生态系统**
- **空白市场**: 跨领域的免费作品匹配协作

## 4. 核心概念

### "匹配作品，而非匹配人"

| 传统方式 | Fugue方式 |
|----------|-----------|
| "寻找插画师" | "寻找与我音乐相配的插画" |
| 以人为单位搜索 | 以作品为单位搜索 |
| 先看简历再看作品 | 先看作品再看创作者 |
| "请帮我画" | "我想用这幅画" |

### 为什么是作品匹配？

1. **降低心理门槛**: 不是评价"人"，而是选择"作品"
2. **提高匹配精度**: 可以直接确认具体的风格和氛围
3. **减少沟通成本**: 不需要"我想要这种风格"的说明
4. **互惠结构**: 双方都是拿出自己的作品，形成平等关系

### 吉祥物

Fugue的吉祥物是一只**戴着耳机、手持画笔的河豚（fugu/河豚）**。
这是Fugue与Fugu（河豚）的文字游戏——象征着音乐与视觉创作的融合。

## 5. 核心流程

### 流程概览

```
[发布作品] → [我想创作什么？] → [推荐匹配的作品]
  30秒         选择项目类型        基于标签的匹配
```

### 步骤1: 发布作品（30秒）

```
粘贴链接 → 自动预览 → 添加标签 → 完成！
```

- 粘贴外部链接（YouTube、SoundCloud、pixiv、Twitter等）
- 自动生成预览（OGP / oEmbed）
- 选择领域（音乐、插画、视频、歌词等）
- 添加标签（氛围、流派、风格等）
- 30秒内完成发布

### 步骤2: "我想创作什么？"

- 选择项目类型模板
  - 例: "音乐MV"、"专辑封面"、"歌曲插画"等
- 根据项目类型，系统了解需要哪些领域的作品

### 步骤3: 推荐匹配的作品

- 基于标签匹配：将用户作品的标签与其他作品的标签进行比对
- 推荐所需领域中与之匹配的作品
- 例: 上传了一首"柔和、钢琴、抒情"的曲子 → 推荐带有"柔和、水彩、抒情"标签的插画

## 6. MVP范围

### 包含的功能（P0）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 作品发布 | 粘贴外部链接，自动预览，添加标签 | P0 |
| 创作者资料 | 昵称、角色、简介、SNS链接、头像 | P0 |
| 推荐/作品匹配 | 基于项目类型模板和标签匹配推荐作品 | P0 |
| 探索 | 按领域/标签/项目类型筛选浏览 | P0 |

### 排除的功能（Post-MVP）

| 功能 | 排除理由 | 时机 |
|------|----------|------|
| 项目看板 | MVP阶段只需SNS联系即可 | 需要合作管理需求后 |
| AI推荐 | 先验证标签匹配是否足够 | 标签匹配不够用时 |
| 支付功能 | 核心价值是免费协作 | 付费需求出现时 |
| 媒体存储 | 利用外部链接降低成本 | 外部链接不足时 |
| 站内消息（DM） | 通过SNS链接替代 | 用户要求站内沟通时 |

## 7. 推荐系统路线图

| 版本 | 方法 | 说明 |
|------|------|------|
| v1 | 标签匹配 | 比较作品标签的交集，按匹配度排序 |
| v2 | 协作历史加权 | 在标签匹配的基础上，为有协作经历的创作者增加权重 |
| v3 | 基于嵌入的跨领域相似度 | 将音乐、插画、视频嵌入到同一向量空间，计算跨领域相似度 |

### v1: 标签匹配（MVP）

```
用户作品标签: [柔和, 钢琴, 抒情, 冬天]
候选作品标签: [柔和, 水彩, 抒情, 雪]
匹配标签: [柔和, 抒情] → 匹配度 2/4 = 50%
```

### v2: 协作历史加权

```
基础分数: 标签匹配度
加分: 过去有过协作的创作者 +0.2
加分: 同一项目类型内的协作 +0.1
```

### v3: 跨领域嵌入

```
音乐嵌入向量 ──┐
                ├── 统一向量空间 → 余弦相似度
插画嵌入向量 ──┘
```

## 8. 冷启动策略

### 策略1: 最小化发布门槛

- 粘贴链接即可完成发布（30秒）
- 不需要上传文件
- 不需要写长篇描述
- 仅靠标签即可形成最基本的信息结构

### 策略2: 供给侧优先

- 先积累作品，再引导匹配需求
- 初期无需双向匹配，单纯浏览也有价值
- 作品本身就是内容，即使没有匹配功能也可浏览

### 策略3: 自己成为第一个用户

- 发布自己的作曲作品
- 实际体验"想找插画"的流程
- 基于真实使用经验进行迭代

### 策略4: 聚焦小众社区

- 初期目标: 音乐MV制作社区
- Vocaloid创作者、独立音乐家、业余插画师
- 从小社区开始，逐步扩展

### 策略5: 维持现有SNS关系

- 不取代Twitter/pixiv，而是作为补充
- 发现在Fugue上完成，联系通过SNS进行
- 降低用户迁移成本

## 9. 关键风险

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 没有用户上传作品 | 致命 | 高 | 冷启动策略执行，自己先使用，先积累供给侧 |
| 标签匹配质量差 | 高 | 中 | v1验证后，若不够用则推进v2/v3路线图 |
| 用户不理解"作品匹配"概念 | 中 | 中 | 引导式UX，项目类型模板让意图更明确 |
| 版权/授权纠纷 | 高 | 低 | "标注出处即可免费使用"作为默认授权，明确标注在作品页面 |
