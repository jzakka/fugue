# Discovery Plan: Fugue — 跨媒体创作策展平台

**日期**: 2026-04-03
**产品阶段**: 新项目（副项目 / 作品集）
**Discovery Question**: 能让所有类型创作内容在一个地方被发现和策展的MVP是什么？

---

## 背景

- 创作内容分散在各平台：音乐在SoundCloud，插画在pixiv，视频在YouTube，代码在GitHub
- 每个平台只处理自己的领域，无法跨领域发现
- 要找到"与这首音乐搭配的插画"需要浏览多个平台
- 现有策展服务（Pinterest、Are.na）以图片为主。音乐/代码是二等公民

## 目标用户

**享受探索和收集各领域创作内容的人**

- 收集灵感的创作者、整理参考的设计师、品味策展爱好者
- 不限领域：音乐、插画、视频、文字、代码、3D、游戏、声音
- 初始利基：对音乐MV制作感兴趣的创作者

## 市场调研

### 定位图

```
              仅图片           跨媒体
                │                │
  有推荐      Pinterest        ▶ Fugue ◀
                │                │
  无推荐      Are.na, Raindrop (空白)
```

### 参考服务

| 服务 | 模式 | 与Fugue的区别 |
|------|------|--------------|
| Pinterest | 图片策展（钉子+看板+推荐） | 仅图片。无法钉音乐/代码 |
| Are.na | 研究/存档策展 | 无推荐。小众社区 |
| Raindrop.io | 书签管理器 | 整理工具，非策展/推荐 |
| Piapro | 创作投稿+许可共享 | 限于Vocaloid生态。无推荐 |

### 核心洞察

- **跨媒体策展+推荐是空白领域**
- 将Pinterest在图片上验证的模式扩展到所有创作媒体
- 跨领域发现创作内容是核心价值

## 核心概念

**Pinterest for all creative media.**

音乐、插画、视频、文字、代码在同一个信息流中共存。点击越多，越能学习你的品味，推荐越好的作品。

## MVP范围

### 包含

1. **钉子** — 外部URL策展。OG自动预览+领域自动检测+标签
2. **看板** — 将钉子按主题分组的集合。公开/私密设置
3. **推荐信息流** — 标签频率启发式（v1）。批量/Redis缓存
4. **隐式品味学习** — 记录view/pin/board_add事件
5. **相关作品** — 作品详情底部展示标签相似作品
6. **认证** — Google/Discord OAuth（已实现）

### 排除

| 功能 | 排除原因 | 替代方案 |
|------|----------|----------|
| 协作匹配 | 专注于策展 | 在平台外自然发生 |
| ML推荐 | 初期数据不足 | 从标签启发式开始 |
| 嵌入式播放器 | OG预览足够 | 跳转外部链接 |

### 推荐路线图

| 阶段 | 方法 | 时机 |
|------|------|------|
| v1 | 标签频率启发式 | MVP |
| v2 | 引入特征存储 | 数据积累后 |
| v3 | ML模型训练 | 基于特征存储 |

## Cold Start策略

1. 最小化钉子门槛（URL+标签，30秒）
2. 自己成为第一个用户
3. 从利基社区（音乐MV创作者）开始
4. 保持现有SNS活动（基于外部链接）
5. 通过SNS分享看板引流

## 关键风险

| 风险 | 说明 | 缓解策略 |
|------|------|----------|
| 转换动机不足 | "为什么要在这里发链接？" | 看板整理+推荐 = 比浏览器书签更好的体验 |
| Cold Start | 作品不足导致推荐失效 | 最小化门槛+自己先用 |
| 标签匹配质量 | 仅靠标签能匹配品味吗？ | v1验证后演进到ML |
| OG fetch限制 | 部分平台OG元数据不全 | 手动输入回退 |
