# PRD: Fugue

---

## 1. 概要

**Fugue**は、創作物ベースのコラボレーションマッチングプラットフォームである。

クリエイターが自分の作品（音楽、イラスト、映像など）を外部リンクで投稿し、タグベースのレコメンドを通じて異分野のクリエイターとコラボ機会を得られるサービスを目指す。

マスコットは、ヘッドフォンをつけて絵筆を持ったフグ。Fugue（フーガ）とフグ（河豚）の語呂合わせである。

## 3. 背景

### コンテキスト

インディー・趣味クリエイターが増加しているが、異分野間のコラボレーションは依然としてTwitter DMなどの非構造的な方法に依存している。

- 作曲はできるがMVを作れない
- イラストを描けるが活用先がない
- 映像編集スキルはあるが素材がない

このようなクリエイターは「お金を払って依頼する」ほどではないが、「一緒に作りたい」というニーズを持っている。

### なぜ今なのか

1. **クリエイターエコノミーの成長** — YouTube、TikTok等でインディークリエイターの活動の場が拡大
2. **AIツールの普及** — 制作ハードルの低下により、趣味クリエイター人口が増加
3. **既存プラットフォームの限界** — 有料コミッションは多いが、無料コラボマッチングの空白地帯が存在
4. **コミュニティ需要** — Twitter/Xでの#コラボ募集ツイートの増加が潜在需要を示唆

## 5. 市場セグメント

### ターゲットユーザー

> 収益よりコラボ機会を求める趣味/インディークリエイター

#### 主要セグメント

| セグメント | 説明 | 規模感 |
|-----------|------|--------|
| インディー作曲家 | DTMで作曲、SoundCloud/YouTube投稿 | 中 |
| 趣味イラストレーター | Pixiv/Twitter投稿、コミッション未経験 | 大 |
| 映像編集者 | After Effects/Premiere Pro使用、素材不足 | 小 |
| ボカロP | 楽曲制作、MV用イラスト・映像を探している | 中 |

### 初期ニッチ

**「MVを作りたい作曲家 × イラストレーター」のマッチング**

- 最もニーズが明確
- 完成物（MV）が分かりやすい
- ボカロ文化圏との親和性が高い

### 除外するセグメント

| セグメント | 除外理由 |
|-----------|----------|
| プロクリエイター | 有料コミッションプラットフォームが既に存在 |
| 企業/商用利用 | 無料コラボモデルと相性が悪い |
| ゲーム開発者 | スコープが広すぎる、専門プラットフォームが存在 |
| 小説家/ライター | テキスト作品のマッチングはv1のスコープ外 |

### 提供価値

|------|------|
| **作品発見** | 自分の作品を異分野のクリエイターに見つけてもらえる |

## 7. ソリューション

### ユーザーフロー

```
┌─────────┐    ┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 会員登録  │───→│ 作品投稿  │───→│ プロジェクト  │───→│ レコメンド │
│          │    │          │    │ タイプ選択    │    │ 閲覧     │
└─────────┘    └──────────┘    └──────────────┘    └──────────┘
                                                        │
                                                        ↓
                                                   ┌──────────┐
                                                   │ 作品詳細  │
                                                   │ → SNSで   │
                                                   │   コンタクト│
                                                   └──────────┘
```

### 機能一覧

#### 機能1: 外部リンクベースの作品投稿

| 項目 | 内容 |
|------|------|
| 概要 | 外部URLを貼り付けて作品を投稿する |
| 投稿フロー | URL入力 → oEmbed/OGPで自動プレビュー → タグ選択 → 説明入力（任意） → 投稿 |
| 対応プラットフォーム | YouTube, SoundCloud, Pixiv, Twitter/X, ニコニコ動画, Bandcamp |
| 自動取得情報 | タイトル、サムネイル、説明、埋め込みプレーヤー |
| タグ | ジャンル、雰囲気、テーマから選択（プリセット + カスタム） |
| 制限 | 1ユーザーあたり最大50作品 (MVP) |

##### 対応oEmbed/OGP

| プラットフォーム | 方式 | 埋め込み |
|-----------------|------|----------|
| YouTube | oEmbed | iframe プレーヤー |
| SoundCloud | oEmbed | iframe プレーヤー |
| Pixiv | OGP | サムネイル + リンク |
| Twitter/X | OGP | サムネイル + リンク |
| ニコニコ動画 | OGP | サムネイル + リンク |
| Bandcamp | oEmbed | iframe プレーヤー |
| その他 | OGPフォールバック | サムネイル + リンク |

#### 機能2: クリエイタープロフィール

| 項目 | 内容 |
|------|------|
| 概要 | クリエイターの基本情報と投稿作品を一覧表示 |
| 必須情報 | ニックネーム、分野（複数選択可）、自己紹介 |
| 任意情報 | プロフィール画像、SNSリンク（Twitter, YouTube, Pixiv等） |
| 表示内容 | プロフィール情報 + 投稿作品グリッド |
| 分野選択肢 | 作曲、イラスト、映像編集、ボーカル、デザイン、その他 |

#### 機能3: レコメンド/作品マッチング

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーの投稿作品に基づいて、コラボ可能な他分野の作品を推薦 |
| アルゴリズム (v1) | タグ一致度ベースのスコアリング |
| 表示タイミング | 作品投稿直後 + ホーム画面 |
| 表示形式 | 「この作品に合うかも」カード一覧 |
| フィルター | 分野、タグ |

##### レコメンドロジック (v1)

```
入力: ユーザーAの作品P (分野: 音楽, タグ: [ローファイ, 夜, チル])

1. 異分野フィルター: P.分野 ≠ 候補.分野
2. タグ一致度計算: score = |P.タグ ∩ 候補.タグ| / |P.タグ ∪ 候補.タグ|
3. スコア降順ソート
4. 上位20件を返却

出力: [
  { 作品: "夜景イラスト", 分野: イラスト, 一致タグ: [夜], スコア: 0.33 },
  { 作品: "チルアニメーション", 分野: 映像, 一致タグ: [チル, 夜], スコア: 0.50 },
  ...
]
```

#### 機能4: 探索

| 項目 | 内容 |
|------|------|
| 概要 | 全作品をフィルター付きで一覧閲覧 |
| フィルター | 分野、タグ、プロジェクトタイプ |
| ソート | 新着順、人気順（いいね数） |
| 表示形式 | グリッドレイアウト、カード型 |
| ページネーション | 無限スクロール (カーソルベース) |

#### 機能5: リアクション

| 項目 | 内容 |
|------|------|
| 概要 | 作品に対する「いいね」とコラボ関心表明 |
| いいね | ハートボタン、トグル型 |
| コラボ関心 | 「一緒に作りたい」ボタン → 相手に通知 |
| 通知 | アプリ内通知（メール通知はPost-MVP） |

### プロジェクトタイプテンプレート

| プロジェクトタイプ | 説明 | 必要分野 | 完成物 |
|-------------------|------|----------|--------|
| MV制作 | 音楽+イラスト+映像で MV作成 | イラスト, 映像編集 | YouTube MV |
| アルバムアート | 音楽+イラストでアルバムカバー作成 | イラスト | アルバムカバー画像 |
| コラボ曲 | 2人以上の作曲家によるコラボ楽曲 | 作曲 | 楽曲 |
| アニメーション | イラスト+映像でアニメーション制作 | 映像編集 | アニメーション動画 |
| 自由コラボ | テンプレートなし、自由形式 | 指定なし | 自由 |

### 前提条件

| 前提 | 説明 | 検証方法 |
|------|------|----------|
| クリエイターは外部リンクで作品を持っている | YouTube、SoundCloud、Pixiv等に作品を既に公開している | ユーザーインタビュー |
| タグマッチングでコラボ相性を判断できる | 同じタグ = 似た雰囲気 = コラボ可能性が高い | A/Bテスト (v2) |
| 無料コラボへの需要がある | お金を払わずにコラボしたいクリエイターが十分にいる | 登録数・投稿数で検証 |
| SNSリンクでコンタクトが取れる | DM機能がなくても、SNSリンク経由でコンタクト可能 | コラボ成立数で検証 |
| クレジット表記ライセンスが受け入れられる | 趣味クリエイターにとって「名前を載せてもらう」は十分な対価 | ユーザーフィードバック |
