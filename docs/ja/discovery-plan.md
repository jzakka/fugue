# Discovery Plan: Fugue — クロスメディア創作キュレーションプラットフォーム

**日付**: 2026-04-03
**製品段階**: 新規（サイドプロジェクト / ポートフォリオ）
**Discovery Question**: あらゆるメディアの創作物を一箇所で発見・キュレーションできるMVPとは？

---

## 背景

- 創作物はプラットフォームごとに散在：音楽はSoundCloud、イラストはpixiv、映像はYouTube、コードはGitHub
- 各プラットフォームは自分の分野のみ扱うため、クロス分野の発見が不可能
- 「この音楽に合うイラスト」を見つけるには複数のプラットフォームを巡る必要がある
- 既存のキュレーションサービス（Pinterest、Are.na）は画像に特化。音楽/コードは二級市民扱い

## ターゲットユーザー

**さまざまな分野の創作物を探索・収集することを楽しむ人**

- インスピレーション収集するクリエイター、リファレンス整理するデザイナー、趣味キュレーター
- 分野不問：音楽、イラスト、映像、文章、コード、3D、ゲーム、サウンド
- 初期ニッチ：音楽MV制作に関心のあるクリエイター

## 市場調査

### ポジショニングマップ

```
              画像のみ           クロスメディア
                │                    │
  推薦あり    Pinterest             ▶ Fugue ◀
                │                    │
  推薦なし    Are.na, Raindrop     (空白)
```

### 参考サービス

| サービス | モデル | Fugueとの違い |
|----------|------|--------------|
| Pinterest | 画像キュレーション（ピン＋ボード＋推薦） | 画像に特化。音楽/コードのピン不可 |
| Are.na | リサーチ/アーカイブキュレーション | 推薦なし。ニッチコミュニティ |
| Raindrop.io | ブックマークマネージャー | 整理ツール。キュレーション/推薦ではない |
| Piapro | 創作投稿＋ライセンス共有 | ボカロエコシステム限定。推薦なし |

### 核心インサイト

- **クロスメディアキュレーション＋推薦は空白地帯**
- Pinterestが画像で証明したモデルを全創作メディアに拡張
- 分野を越えた創作物の発見が核心価値

## 核心コンセプト

**Pinterest for all creative media.**

音楽、イラスト、映像、文章、コードが一つのフィードに共存する。クリックするほど好みを学習し、より良い作品が表示される。

## MVPスコープ

### 含む

1. **ピン** — 外部URLキュレーション。OG自動プレビュー＋分野自動検出＋タグ
2. **ボード** — ピンをテーマ別にまとめるコレクション。公開/非公開設定
3. **推薦フィード** — タグ頻度ヒューリスティック（v1）。バッチ/Redisキャッシュ
4. **暗黙的嗜好学習** — view/pin/board_addイベント記録
5. **関連作品** — 作品詳細下部にタグ類似作品表示
6. **認証** — Google/Discord OAuth（実装済み）

### 除外

| 機能 | 除外理由 | 代替手段 |
|------|----------|----------|
| コラボレーションマッチング | キュレーションに集中 | プラットフォーム外で自然に |
| ML推薦 | 初期データ不足 | タグヒューリスティックで開始 |
| 埋め込みプレイヤー | OGプレビューで十分 | 外部リンクへ遷移 |

### 推薦ロードマップ

| 段階 | 方法 | 時期 |
|------|------|------|
| v1 | タグ頻度ヒューリスティック | MVP |
| v2 | フィーチャーストア導入 | データ蓄積後 |
| v3 | MLモデル学習 | フィーチャーストア基盤 |

## Cold Start戦略

1. ピンの障壁最小化（URL＋タグ、30秒）
2. 自分が最初のユーザー
3. ニッチコミュニティ（音楽MVクリエイター）から開始
4. 既存SNS活動を維持（外部リンクベース）
5. ボード共有でSNSから流入

## 主要リスク

| リスク | 説明 | 緩和策 |
|--------|------|--------|
| 転換動機不足 | 「なぜここにリンクを？」 | ボード整理＋推薦 = ブックマークより良い体験 |
| Cold Start | 作品不足で推薦不可 | 障壁最小化＋自分が最初のユーザー |
| タグマッチング品質 | タグだけで好みを合わせられるか | v1で検証後、ML進化 |
| OG fetch制限 | 一部プラットフォームのOG不備 | 手動入力フォールバック |
