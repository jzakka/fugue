## ADDED Requirements

### Requirement: 외부 URL을 핀으로 저장한다
시스템은 인증된 유저가 외부 창작물 URL을 핀으로 저장할 수 있어야 한다(SHALL). 핀은 소유권 주장이 아닌 큐레이션이다.

#### Scenario: 유효한 핀 생성
- **WHEN** 인증된 유저가 URL, 분야, 태그를 포함하여 핀 생성을 요청하면
- **THEN** 핀이 생성되고 생성된 핀 정보가 반환된다

#### Scenario: URL은 필수이다
- **WHEN** URL 없이 핀 생성을 요청하면
- **THEN** 유효성 검사 오류가 반환된다

#### Scenario: 분야는 필수이다
- **WHEN** 분야 없이 핀 생성을 요청하면
- **THEN** 유효성 검사 오류가 반환된다

#### Scenario: 태그는 1개 이상 5개 이하이다
- **WHEN** 태그가 없거나 5개를 초과하여 핀 생성을 요청하면
- **THEN** 유효성 검사 오류가 반환된다

#### Scenario: 같은 URL을 여러 유저가 핀할 수 있다
- **WHEN** 이미 다른 유저가 핀한 URL로 핀 생성을 요청하면
- **THEN** 정상적으로 핀이 생성된다 (URL 유니크 제약 없음)

#### Scenario: 미인증 유저는 핀을 생성할 수 없다
- **WHEN** 미인증 상태에서 핀 생성을 요청하면
- **THEN** 인증 오류가 반환된다

---

### Requirement: 핀 상세 정보를 조회한다
시스템은 핀의 상세 정보를 핀한 유저 정보와 함께 제공해야 한다(SHALL).

#### Scenario: 유효한 핀 조회
- **WHEN** 존재하는 핀 ID로 조회하면
- **THEN** 핀 상세 정보와 핀한 유저의 닉네임, 아바타가 포함된 응답이 반환된다

#### Scenario: 존재하지 않는 핀
- **WHEN** 존재하지 않는 핀 ID로 조회하면
- **THEN** 404 응답이 반환된다

---

### Requirement: 본인의 핀을 삭제한다
시스템은 인증된 유저가 자신이 만든 핀만 삭제할 수 있어야 한다(SHALL).

#### Scenario: 본인 핀 삭제
- **WHEN** 인증된 유저가 자신의 핀 삭제를 요청하면
- **THEN** 핀이 삭제된다

#### Scenario: 타인의 핀 삭제 불가
- **WHEN** 인증된 유저가 다른 유저의 핀 삭제를 요청하면
- **THEN** 해당 핀을 찾을 수 없다는 응답이 반환된다

---

### Requirement: 핀 목록을 필터링하여 조회한다
시스템은 분야와 태그로 필터링된 핀 목록을 페이지네이션하여 제공해야 한다(SHALL).

#### Scenario: 분야 필터
- **WHEN** 특정 분야로 핀 목록을 요청하면
- **THEN** 해당 분야의 핀만 반환된다

#### Scenario: 태그 필터
- **WHEN** 태그로 핀 목록을 요청하면
- **THEN** 해당 태그와 겹치는 핀이 반환된다

#### Scenario: 최신순 정렬
- **WHEN** 필터 없이 핀 목록을 요청하면
- **THEN** 생성 시각 역순으로 정렬되어 반환된다

#### Scenario: 유저별 필터
- **WHEN** 특정 유저 ID로 핀 목록을 요청하면
- **THEN** 해당 유저가 핀한 작품만 반환된다

### Requirement: URL의 OG 메타데이터를 가져온다
시스템은 외부 URL의 OG 메타데이터를 서버 사이드에서 추출하여 제공해야 한다(SHALL).

#### Scenario: OG 메타데이터 추출 성공
- **WHEN** 유효한 URL로 메타데이터 조회를 요청하면
- **THEN** 제목, 설명, 썸네일 이미지, 사이트명이 반환된다

#### Scenario: 도메인 기반 분야 자동감지
- **WHEN** URL의 도메인이 알려진 창작 플랫폼이면
- **THEN** 감지된 분야가 함께 반환된다

#### Scenario: OG 태그가 없는 URL
- **WHEN** OG 메타데이터가 없는 URL로 요청하면
- **THEN** URL만 포함된 부분 응답이 반환되고, 유저가 직접 입력할 수 있다

#### Scenario: 접근 불가능한 URL
- **WHEN** 접근할 수 없거나 응답 시간이 초과된 URL로 요청하면
- **THEN** 오류 메시지와 함께 부분 응답이 반환된다

#### Scenario: 내부 네트워크 접근 차단
- **WHEN** 내부 네트워크 주소로 메타데이터 조회를 요청하면
- **THEN** 지원되지 않는 URL이라는 오류가 반환된다

#### Scenario: 과도한 요청 제한
- **WHEN** 짧은 시간 내 과도한 메타데이터 조회 요청이 발생하면
- **THEN** 요청이 제한된다
