## ADDED Requirements

### Requirement: 보드를 생성한다
시스템은 인증된 유저가 핀을 묶는 보드를 생성할 수 있어야 한다(SHALL).

#### Scenario: 보드 생성
- **WHEN** 인증된 유저가 보드 이름을 포함하여 생성을 요청하면
- **THEN** 보드가 생성되고 기본 공개 상태로 설정된다

#### Scenario: 보드 이름은 필수이다
- **WHEN** 이름 없이 보드 생성을 요청하면
- **THEN** 유효성 검사 오류가 반환된다

---

### Requirement: 보드를 조회한다
시스템은 보드의 상세 정보와 포함된 핀 목록을 제공해야 한다(SHALL).

#### Scenario: 공개 보드 조회
- **WHEN** 누구나 공개 보드를 조회하면
- **THEN** 보드 정보와 핀 목록이 반환된다

#### Scenario: 비공개 보드는 소유자만 조회
- **WHEN** 소유자가 아닌 유저가 비공개 보드를 조회하면
- **THEN** 보드를 찾을 수 없다는 응답이 반환된다

#### Scenario: 소유자는 비공개 보드 조회 가능
- **WHEN** 소유자가 자신의 비공개 보드를 조회하면
- **THEN** 보드 정보와 핀 목록이 반환된다

---

### Requirement: 보드를 수정한다
시스템은 보드 소유자만 보드를 수정할 수 있어야 한다(SHALL).

#### Scenario: 소유자가 보드 이름 수정
- **WHEN** 소유자가 보드 이름 변경을 요청하면
- **THEN** 보드 이름이 업데이트된다

#### Scenario: 소유자가 공개 여부 변경
- **WHEN** 소유자가 보드의 공개 설정을 변경하면
- **THEN** 공개 여부가 업데이트된다

#### Scenario: 소유자가 아니면 수정 불가
- **WHEN** 소유자가 아닌 유저가 보드 수정을 요청하면
- **THEN** 권한 오류가 반환된다

---

### Requirement: 보드를 삭제한다
시스템은 보드 소유자만 보드를 삭제할 수 있어야 한다(SHALL).

#### Scenario: 소유자가 보드 삭제
- **WHEN** 소유자가 보드 삭제를 요청하면
- **THEN** 보드와 보드-핀 관계가 삭제된다 (핀 자체는 삭제되지 않는다)

---

### Requirement: 보드에 핀을 추가/제거한다
시스템은 보드 소유자가 자신의 보드에 핀을 추가하거나 제거할 수 있어야 한다(SHALL).

#### Scenario: 보드에 핀 추가
- **WHEN** 소유자가 보드에 핀 추가를 요청하면
- **THEN** 핀이 보드에 추가된다

#### Scenario: 같은 핀 중복 추가 방지
- **WHEN** 이미 보드에 있는 핀을 다시 추가 요청하면
- **THEN** 중복 추가 오류가 반환된다

#### Scenario: 하나의 핀이 여러 보드에 속할 수 있다
- **WHEN** 동일한 핀을 서로 다른 보드에 추가하면
- **THEN** 두 보드 모두에서 해당 핀이 표시된다

#### Scenario: 보드에서 핀 제거
- **WHEN** 소유자가 보드에서 핀 제거를 요청하면
- **THEN** 보드-핀 관계가 삭제된다 (핀 자체는 유지된다)

---

### Requirement: 유저의 보드 목록을 조회한다
시스템은 유저의 보드 목록을 제공해야 한다(SHALL).

#### Scenario: 본인의 보드 목록
- **WHEN** 인증된 유저가 자신의 보드 목록을 조회하면
- **THEN** 공개 + 비공개 모든 보드가 반환된다

#### Scenario: 타인의 보드 목록
- **WHEN** 다른 유저의 보드 목록을 조회하면
- **THEN** 공개 보드만 반환된다
